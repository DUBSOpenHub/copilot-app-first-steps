# Contributing to Copilot App First Steps

Thanks for helping make the GitHub Copilot app easier for first-time users.

## Core principles

1. Beginner-first: assume no technical background.
2. One obvious start: do not add competing first steps.
3. One click first: keep terminal/manual setup out of the primary path.
4. Safety before action: explain approvals before editing workflows.
5. Text equivalents: every visual cue needs a text explanation.

## Workflow

1. Fork the repo and create a branch.
2. Make the smallest change that solves the problem.
3. Update `TESTING.md` and eval tasks when behavior changes.
4. Keep all `SKILL.md` copies synchronized.
5. Run the validation workflow before opening a PR.
6. Fill out the pull request template.

## Modifying the tutor

- Keep prompts plain and friendly.
- Avoid unexplained Git or developer jargon.
- Prefer copy/paste prompt recipes.
- Do not use destructive examples.
- Do not claim one-click skill install exists unless the app supports it.

## Reporting issues

Use GitHub Issues for bugs, feature requests, and lesson ideas. Security issues must follow `SECURITY.md`.
