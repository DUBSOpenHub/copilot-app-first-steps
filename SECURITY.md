# Security Policy

## Supported versions

| Version | Supported |
| --- | --- |
| v0.1.x | Yes |
| < v0.1 | No |

## Reporting a vulnerability

Do not open a public GitHub issue for security vulnerabilities.

If you discover a security issue in Copilot App First Steps, report it privately.

Preferred channel:

- GitHub Security Advisories, if enabled on the repository.

Alternative channel:

- Email: `security@dubsopenhub.com`

## What to include

- A clear description of the issue.
- Steps to reproduce.
- Potential impact.
- Safe proof-of-concept details, if available.

## Response timeline

- Acknowledgment: within 24 hours.
- Assessment: within 72 hours.
- Resolution: depends on severity.

## Scope

In scope:

- Prompt-injection risks in the tutor instructions.
- Unsafe onboarding guidance.
- Incorrect permission or approval guidance.
- One-click install page behavior.
- Leaks or accidental collection of learner information.

Out of scope:

- Vulnerabilities in the GitHub Copilot app itself.
- Vulnerabilities in GitHub.com or underlying LLM providers.
- User-generated prompts outside this tutor's guidance.
